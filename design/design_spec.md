# プロジェクト詳細設計書：デッキ構築型マインスイーパ「わためは悪くないよねぇ」

## 1. 開発環境・技術スタック

- **ゲームエンジン**: Phaser 3 (JavaScript)
- **インフラ**: Docker / Docker Compose (Nginx alpine)
- **言語**: HTML5, CSS3, JavaScript (ES6+)

## 2. ディレクトリ構成

```
project-root/
├── docker-compose.yml      # Docker構成定義
├── Dockerfile              # Nginxイメージビルド定義
├── public/
│   ├── index.html          # エントリポイント
│   ├── style.css           # 画面レイアウト・キャンバス配置
│   └── assets/             # 画像（セル、カード、エフェクト）、SE、BGM
└── src/
    ├── main.js             # Phaser構成設定・シーン管理
    ├── scenes/             # 各ゲーム画面（Boot, Menu, Game, UI）
    ├── entities/           # ロジック（Board, CardManager, Jamming）
    └── utils/              # 共通定数・計算ヘルパー
```

## 3. シーン詳細設計

### 3.1 BootScene (プリロード)

**役割**: 全アセットのメモリロード。

処理フロー:
- セル（0-8, 地雷, 旗, カード）、カードUI、背景、演出用画像を読み込む。
- ロード完了後、MenuScene へ遷移。

### 3.2 MenuScene (メインメニュー)

**役割**: ゲーム開始の待機と初期化。

処理フロー:
- スタートボタンを表示。
- プレイヤーの初期状態（Stage: 1, HP: 1, 手札: 空）を定義。

### 3.3 GameScene (メインロジック)

**役割**: 盤面描画、入力検知、ジャミング判定、勝敗判定。

主要メソッド:
- `setupStage(level)`: STAGE定数から盤面サイズ、地雷数、妨害率を設定。
- `handleCellClick(x, y)`: クリック座標を取得し、Jammingクラスへ処理を委譲。

## 4. クラス別ロジック詳細

### 4.1 Board.js (盤面管理)

- **データ構造**: Cell オブジェクトの2次元配列。
- **プロパティ**: `x`, `y`, `isMine`, `isCard`, `isRevealed`, `isFlagged`, `neighborMines`

生成ロジック:
- 指定サイズで配列を初期化。
- 地雷をランダムに配置。
- 地雷のないマスに `isCard` フラグをランダム配置。
- 各セルの周囲8マスを走査し `neighborMines` を計算。

メソッド:
- `reveal(x, y)`: 指定座標を開封。数字0なら周囲を再帰開封。
- `moveMine(x, y)`: 対象の地雷を未開封の別マスへ移し、盤面の数字を再計算。

### 4.2 Jamming.js (妨害ロジック)

- **判定**: `Math.random()` がステージ妨害率以下なら発動。
- **座標スリップ**: 発動時、クリック座標 `(x, y)` を周囲 1〜3 マス分ランダムにずらした `(x', y')` を生成。
- **演出**: 画面中央に「わためは悪くないよねぇ」のメッセージとエフェクトを表示。

### 4.3 CardManager.js (デッキ・リソース管理)

- **手札管理**: 最大4枚まで保持。
- **カード獲得 (`addCard`)**: 手札が満杯時に新カードを得た場合、交換UIを表示。

カード効果:
- **マインサーチ1/3**: 未発見の地雷を特定し旗を立てる。
- **リヴァイブ**: 地雷爆発を自動で無効化。
- **マインムーブ**: 地雷の場所を移動。
